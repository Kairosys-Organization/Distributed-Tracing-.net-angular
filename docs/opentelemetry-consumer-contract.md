# OpenTelemetry Consumer Contract

This document outlines the OpenTelemetry Protocol (OTLP) contract for consuming traces generated by both the **.NET Backend** and the **Angular Frontend**.

The OpenTelemetry Collector standardizes data from both sources into a unified contract, allowing your external trace consumer (e.g., Python service) to receive the data seamlessly via either **gRPC** or **REST (HTTP/JSON)**.

---

## 1. Transport Contracts

### A. gRPC Consumer (Recommended)
This approach consumes traces over an open gRPC bidirectional stream utilizing Protobuf serialization.

- **Protocol**: HTTP/2 (gRPC)
- **Service Name**: `opentelemetry.proto.collector.trace.v1.TraceService`
- **RPC Method**: `Export`
- **Request Message**: `ExportTraceServiceRequest`
- **Response Message**: `ExportTraceServiceResponse` (Always return an empty response)

#### Python gRPC Example
```python
import grpc
from concurrent import futures
from opentelemetry.proto.collector.trace.v1 import trace_service_pb2_grpc, trace_service_pb2

class OTLPTraceReceiver(trace_service_pb2_grpc.TraceServiceServicer):
    def Export(self, request, context):
        # request is an ExportTraceServiceRequest protobuf object
        for resource_span in request.resource_spans:
            for scope_span in resource_span.scope_spans:
                for span in scope_span.spans:
                    # Look for exception events inside the span
                    for event in span.events:
                        if event.name == "exception":
                            for attr in event.attributes:
                                if attr.key == "exception.stacktrace":
                                    print(f"Stack Trace Found:\n{attr.value.string_value}")
                                    
        return trace_service_pb2.ExportTraceServiceResponse()

server = grpc.server(futures.ThreadPoolExecutor(max_workers=10))
trace_service_pb2_grpc.add_TraceServiceServicer_to_server(OTLPTraceReceiver(), server)
server.add_insecure_port('[::]:4317')
server.start()
server.wait_for_termination()
```

### B. REST (HTTP/JSON) Consumer
If your consumer service runs an HTTP framework (like Flask/FastAPI in Python or Express in Node.js), it can receive traces as JSON POST requests.

- **Protocol**: HTTP/1.1 or HTTP/2
- **Path**: `/v1/traces` (This path is standard for OTLP/HTTP)
- **Content-Type**: `application/json`

#### Example JSON Payload Structure:
The JSON schema directly mirrors the Protocol Buffers structure:
```json
{
  "resourceSpans": [
    {
      "resource": {
        "attributes": [
          {
            "key": "service.name",
            "value": { "stringValue": "pathfinder-api" }
          }
        ]
      },
      "scopeSpans": [
        {
          "scope": {
            "name": "OpenTelemetry.Instrumentation.AspNetCore"
          },
          "spans": [
            {
              "traceId": "5b8aa5a2d2c8646c14e4d97e98823166",
              "spanId": "517900b95eb7ac24",
              "name": "GET /api/weatherforecast",
              "kind": 2, 
              "startTimeUnixNano": "1645600000000000000",
              "endTimeUnixNano": "1645600005000000000",
              "attributes": [
                {
                   "key": "http.method",
                   "value": { "stringValue": "GET" }
                }
              ],
              "events": [],
              "status": { "code": 1 }
            }
          ]
        }
      ]
    }
  ]
}
```

#### Python FastAPI Example (REST)
```python
from fastapi import FastAPI, Request

app = FastAPI()

@app.post("/v1/traces")
async def receive_traces(request: Request):
    payload = await request.json()
    resource_spans = payload.get("resourceSpans", [])
    
    for resource_span in resource_spans:
        service_name = "unknown"
        # Extract service name
        for attr in resource_span.get("resource", {}).get("attributes", []):
            if attr["key"] == "service.name":
                service_name = attr["value"]["stringValue"]
                
        # Process spans
        for scope_span in resource_span.get("scopeSpans", []):
            for span in scope_span.get("spans", []):
                print(f"[{service_name}] Span: {span.get('name')}")
                
                # Look for exception events inside the JSON span object
                for event in span.get("events", []):
                    if event.get("name") == "exception":
                        for attr in event.get("attributes", []):
                            if attr.get("key") == "exception.stacktrace":
                                # Extract the stringValue from the nested value object
                                stack_trace = attr.get("value", {}).get("stringValue", "")
                                print(f"Stack Trace Found:\n{stack_trace}")
                
    return {} # Note: OTLP expects an empty JSON response
```

## 2. Shared Data Structure (Resource → Scope → Span)

Regardless of gRPC or REST, the data is heavily nested into three overarching layers:

1. **`ResourceSpans`**: Identifies the service that emitted the traces.
   - Look for the `service.name` attribute (`pathfinder-api`, `pathfinder-ui`, `pathfinder-ui-zoneless`).
2. **`ScopeSpans`**: Identifies the tracing library/instrumentation logic.
   - .NET example: `OpenTelemetry.Instrumentation.AspNetCore`
   - Angular example: `@opentelemetry/instrumentation-document-load`
3. **`Spans`**: The specific measurements (requests, queries, DOM changes).

## 3. Emitted Data Contracts Specific to App Types

### A. .NET Backend Traces (`pathfinder-api`)
The .NET backend emits `SPAN_KIND_SERVER` (for receiving HTTP requests) and `SPAN_KIND_CLIENT` (for making HTTP or DB queries).

#### Attributes to expect (`span.attributes`):
- `http.method`: `GET`, `POST`
- `http.target`: `/api/weatherforecast`
- `http.status_code`: `200`, `500`

#### Errors and Exceptions Contract
According to `Program.cs`, whenever an unhandled exception occurs, the `.NET` API logs it under the OTLP **events** array, NOT attributes. The event must be parsed like so:

- **Condition for failure**: `span.status.code == 2` (STATUS_CODE_ERROR).
- **Events Array**: Look for an event named `exception`.
  - Event Attribute: `exception.type` (e.g., `System.NullReferenceException`)
  - Event Attribute: `exception.message` (e.g., "Object reference not set...")
  - Event Attribute: `exception.stacktrace` (e.g., raw stacktrace string)

*(In Python JSON, this looks like:)*
```json
"events": [
  {
    "name": "exception",
    "timeUnixNano": "1645600000000000000",
    "attributes": [
      { "key": "exception.type", "value": { "stringValue": "System.Exception" } },
      { "key": "exception.message", "value": { "stringValue": "Something went wrong" } },
      { "key": "exception.stacktrace", "value": { "stringValue": "..." } }
    ]
  }
]
```

### B. Angular Frontend Traces (`pathfinder-ui`)
The Angular frontend emits `SPAN_KIND_CLIENT` spans (outbound HTTP requests to the .NET API) and `SPAN_KIND_INTERNAL` spans (UI routing, Document load).

#### Attributes to expect (`span.attributes`):
- `http.method`: HTTP method of the API call.
- `http.url`: Full Angular URL or API destination URL.
- `user_agent.original`: The customer's browser info.
- `document.title`: Current page title.

**Note on connecting Front/Back traces (Distributed Context):** 
When Angular makes an HTTP request to .NET, it attaches `traceparent` HTTP headers to the outbound request. The .NET API automatically receives this header and binds its own execution to the exact same `trace_id`. The Python trace consumer simply groups the UI spans and API spans together using their shared `trace_id` rather than treating them as separate transactions.
